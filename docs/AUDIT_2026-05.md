## Clean dimensions (no action needed)

- Service-layer isolation — no Firebase SDK imports in `src/screens/**`, `src/components/**`, `src/hooks/**`, `src/context/**` outside type-only. Only type-only Firebase import is `src/hooks/useAuth.ts:2`.
- Transactional game writes — 32 `runTransaction` call sites across `games.create.ts`, `games.judge.ts`, `games.turns.ts`, `games.match.ts`.
- No `as any`, `@ts-ignore`, `@ts-expect-error`, or `console.log` in production code. One justified `eslint-disable` at `src/components/VideoRecorder.tsx:68`.
- No Cloud Functions. No `functions/` directory; `pr-gate.yml` enforces.
- No state-management or UI-component libraries in `package.json`.
- Routing: all `<Route>` elements in `src/App.tsx` (one drift noted in F3).
- Storage rules (`storage.rules`): content-type + size + filename + UID binding + append-only + default deny.
- Firestore indexes (`firestore.indexes.json`) aligned with queries in `src/services/games.subscriptions.ts:29–44` and `src/services/clips.ts`.
- Secrets hygiene clean: no committed secrets; `.gitignore` covers `.env*`; `.env.example` placeholders only.
- Build config: `vite.config.ts` `sourcemap: "hidden"`; Firebase + React chunks split.
- Capacitor: no hardcoded prod URLs; `cleartext: true` gated to dev-server-only.
- App Check (`src/firebase.ts:122–230`): opt-in implementation (DeviceCheck / Play Integrity / ReCAPTCHA v3) with Sentry breadcrumbs on init failure. (Currently disabled — see F5.)
- Coverage thresholds (`vite.config.ts:100–110`): `src/services/**` and `src/hooks/**` at 100%; `src/firebase.ts` at 93%/80%; components/screens at 80%/75%.
- CI gates: `main.yml`'s `build-and-test` job runs the full pipeline (`tsc -b && lint && test:coverage && build && check:test-dup`) plus the `e2e` and `lighthouse` jobs. `pr-gate.yml` runs the policy checks separately (`enforce-pr-policy`, `guard-as-any-casts`, `guard-todo-fixme-hack`, `verify-no-cloud-functions`, `verify-workflow-changes`, `check-file-length`, `check-test-duplication`, and `Validate Firebase rules changes`) — it does NOT re-run the full build/test pipeline; that's `main.yml`'s job. `firebase-rules-deploy.yml` validates rules against emulator before deploy. 26 rules-tests files; 4 Playwright specs at audit time (+5 added via F8).

**Correction.** A draft audit agent flagged `games.create.ts`, `games.judge.ts`, `games.turns.ts` as untested. That is wrong — `src/services/games.ts` is a 22-line barrel and `src/services/__tests__/games.test.ts` (1938 LOC) imports through it and exercises `createGame`, `acceptJudgeInvite`, `declineJudgeInvite`, `callBSOnSetTrick`, `judgeRuleSetTrick`, `forfeitExpiredTurn`. The monolith is its own finding (F4).
